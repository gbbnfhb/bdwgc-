## コレクターを構築する

まだ行っていない場合は、コレクターを解凍し、新しく作成した
ディレクトリに次のものがあります:

```sh
tar xvfz gc-<version>.tar.gz
cd gc-<version>
```

コレクターはプライベート ディレクトリに設定、構築、インストールできます。
次のコマンドを使用して、/home/xyz/gc と言います。

```sh
./configure --prefix=/home/xyz/gc --disable-threads
make
make check
make install
```

ここでの`make check`コマンドはオプションですが、強くお勧めします。走ります
基本的な正確性テストには通常 1 分もかかりません。

## プログラムを書く

`gc.h`ファイルをすべてのファイルの先頭に含める必要があります。
ガベージ コレクターを通じてメモリを割り当てます。どこにいても`GC_MALLOC`を呼び出します
`malloc`を呼び出すことになります。これにより、`calloc`のようにメモリがゼロに初期化されます。
結果を明示的にクリアする必要はありません。

オブジェクトにガベージ コレクションされたオブジェクトへのポインターが含まれないことがわかっている場合は、
ヒープを初期化する必要がない場合は、`GC_MALLOC_ATOMIC`を呼び出します。
その代わり。

関数`GC_FREE`が提供されていますが、呼び出す必要はありません。非常に小さい場合
オブジェクトを呼び出さない場合、プログラムのパフォーマンスはおそらく向上します。
コレクターにその仕事を任せてください。

`GC_REALLOC`関数は、C ライブラリ`realloc`と同様に動作します。割り当てます
元のオブジェクトが割り当てられていた場合、初期化されていないポインタのないメモリ
方法。

次のプログラム`loop.c`は簡単な例です。

```c
#include "gc.h"

#include <assert.h>
#include <stdio.h>

int main(void) {
  int i;

  GC_INIT();
  for (i = 0; i < 10000000; ++i) {
    int **p = GC_NEW(int *);
    int *q = GC_NEW_ATOMIC(int);
    assert(*p == 0);
    *p = (int *)GC_REALLOC(q, 2 * sizeof(int));
    if (i % 100000 == 0)
      printf("Heap size = %lu bytes\n",
             (unsigned long)GC_get_heap_size());
  }
  return 0;
}
```


### システム malloc との対話

通常は、ガベージ コレクションによる割り当てとシステムを混同しないことが最善です
`malloc`-`free`。そうした場合、ポインタを保存しないように注意する必要があります。
システム`malloc`で割り当てられたメモリ内のガベージ コレクションされたヒープにコピーされます。

### 他のプラットフォーム

## コンパイルとリンク

上記のアプリケーション`loop.c`テスト プログラムは、次のようにコンパイルしてリンクできます。

```sh
cc -I/home/xyz/gc/include loop.c /home/xyz/gc/lib/libgc.a -o loop
```

`-I`オプションは、コンパイラーを正しいインクルード ディレクトリに指示します。この中で
この場合、静的ライブラリをコンパイル行に直接リストします。ダイナミックな
動的に調整すれば、代わりにライブラリを使用することもできたでしょう。
ローダーを使用してそれを見つけます。例:`LD_LIBRARY_PATH`を設定することによって。
