## トポロジー的なファイナライゼーションの順序を回避する

特定の状況では、ファイナライズ可能なオブジェクト間のサイクルが発生します。
本当に避けられない。最も注目すべき点は、C++ コンパイラーが自己サイクルを導入していることです。
継承を表すため。`GC_register_finalizer_ignore_self`は、
コレクターのファイナライズ部分が自己サイクルを無視するようになりました。これは、
C++ インターフェイス。

Finalize.c には、実際には、意図的に文書化されていないメカニズムが含まれています。
ユーザー定義の依存関係を持つファイナライズ可能なオブジェクトを登録します。問題
この依存関係情報は、メモリ再利用ではなくメモリ再利用にも使用されるということです。
注文を確定するだけです。したがって、誤用すると、たとえ
ファイナライズによって何も作成されない場合。ポインタがぶら下がっている可能性があるリスクは次のとおりです。
`-D JAVA_FINALIZATION`を使用してコレクタを構築することで排除されます。これにより、
この依存関係にもかかわらず、マークされるファイナライザーから到達可能なオブジェクト
はファイナライゼーションの順序としては考慮されません。
