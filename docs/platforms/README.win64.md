AMD64/Intel EM64T (x64) 上の 64 ビット Windows がサポートされています。  コレクターになれるのは、
Microsoft Visual C++ 2005 または mingw-w64 gcc でビルドされています。

この環境では NT_MAKEFILE が使用されています。  タイプ
Visual C++ コマンド ラインの「nmake -f NT_MAKEFILE cpu=AMD64 nodebug=1」
スレッドを使用してダイナミック ライブラリのリリース バリアントを構築するためのウィンドウ
サポート。
コレクターが少なくともある程度機能していることを確認するには、次のようにします。
「nmake -f NT_MAKEFILE cpu=AMD64 check」と入力して、通常のテストをビルドして実行します
プログラム。  これにより、数秒後に gctest.gc.log が作成されます。

動的 GC リンクの場合、cpptest.exe が正しく実行されない可能性があります。  （そうみたいです
場合によっては、演算子 new/delete の間違ったインスタンスが取得されることがあります。)

このプロセスは NT_MAKEFILE の使用法と完全に似ています。
32 ビット ライブラリ バージョンの場合。

NT_MAKEFILE を使用した同様の手順は、静的ファイルの構築にも適用できます。
ライブラリ - 「enable_static=1」を追加の引数として nmake に渡すだけです。
必要に応じて、スレッドなしでライブラリを構築することもできます。
サポート - これは、「disable_threads=1」引数を nmake に渡すことで実行できます。

一部の警告はメイクファイルで明示的にオフになっていることに注意してください。

VC++ 注: 警告を抑制するには、「-D _CRT_SECURE_NO_DEPRECATE」が使用されます。
