# ガベージコレクターをリークディテクタとして使用する

ガベージ コレクターは漏れ検出器として使用できます。この場合、
コレクターの主な機能は、割り当てられたオブジェクトを報告することです。
(通常は`GC_MALLOC`を使用)、割り当ては解除されません (通常は`GC_FREE`を使用)、
にはアクセスできなくなりました。オブジェクトにはアクセスできなくなったため、
通常、後でオブジェクトの割り当てを解除する方法はありません。したがって、それはできます
オブジェクトが「漏洩」したと考えて間違いありません。

これは、単純にカウントするリークディテクタとは大きく異なります。
割り当てられたすべてのオブジェクトが最終的に割り当て解除されることを確認します。
ガベージコレクタベースのリークディテクタは、より正確な情報を提供できます。
オブジェクトが漏洩したときの情報。さらに重要なのは、報告しないことです
「永続的な」データの一部であるため、割り当てが解除されることのないオブジェクト
構造物。したがって、プロセス時にすべてのオブジェクトの割り当てを解除する必要はありません。
終了時間、多くの場合大量のトリガーとなる、役に立たない可能性のあるアクティビティ
ページングの。

ガベージ コレクタは、リーク検出サポートを提供します (コレクタが
`NO_FIND_LEAK`マクロが定義されてビルドされます)。これには以下が含まれます
特徴：

1. リーク検出モードは、実行時に`GC_set_find_leak(1)`によって開始できます。
コレクタをビルドする代わりに、プログラムの起動時に呼び出します。
`FIND_LEAK`マクロが定義されました。

2. 漏洩したオブジェクトは報告され、正しくガベージ コレクションされる必要があります。

コレクタを漏れ検出器として使用するには、次の手順を実行します。

1. 前述のように漏れ検出モードをアクティブにします。

2. すべての割り当てと割り当て解除が完了するようにプログラムを変更します。
ガベージコレクター。

3. 適切な時点で`GC_gcollect`(または`CHECK_LEAKS()`) を呼び出すように手配します。
漏れをチェックするために。 (これは暗黙的に起こりますが、おそらくそうではありません
長時間実行されるプログラムには十分な頻度です)。

2 番目のステップは通常、`-D REDIRECT_MALLOC`を使用して実行できます。
コレクターのビルド時、または`malloc`の定義によるコンパイラー オプション、
`calloc`、`realloc`、`free`(および`posix_memalign`、`reallocarray`、
`reallocf`、`strdup`、`strndup`、`wcsdup`、BSD`memalign`、GNU`valloc`、
GNU`pvalloc`、`freezero`、`freezeroall`) を使用して、対応するガベージを呼び出します。
コレクター機能。しかし、これだけではあまり有益な情報は得られません
コレクターは診断に関する情報を追跡しないため、
オブジェクトがどのように割り当てられたか。エラーレポートにはオブジェクトのみが含まれます
アドレス。

より正確なエラーレポートを得るには、できるだけ多くのプログラムで以下を使用する必要があります。
`GC_DEBUG`を定義した後のこれらの関数のすべて大文字のバリアント、および
次に`gc.h`ファイルを含めます。この環境では、`GC_MALLOC`はマクロです。
少なくとも割り当てポイントのファイル名と行番号が次のようになります。
オブジェクトの一部として保存されます。リークレポートにはこれも含まれます
情報。

多くのコレクター機能 (ファイナライゼーションやリンクの消滅など) が少なくなります。
このコンテキストでは便利ですが、完全にはサポートされていません。それらの使用は通常、
コレクター自体がいくつかの情報をドロップするため、追加の偽のリーク レポートを生成します。
関連するオブジェクト。

一般に、マルチスレッドのサポートについても同じことが当てはまります。ただし、正しいのは、
リーク レポートは、少なくとも LinuxThreads を使用して生成する必要があります。

いくつかのプラットフォーム (現在 Linux/i686、Linux/x86_64、および SPARC) では、`GC_MALLOC`
また、その呼び出しスタックに関するいくつかの追加情報が
物体。このような情報はエラー レポートで再現されます。
非シンボリック形式ですが、デバッガを使用すると非常に便利です。
