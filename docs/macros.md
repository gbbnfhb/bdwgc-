コレクターは、次の目的で大量の条件付きコンパイルを使用します。
プラットフォームの依存関係に対処します。  これは多くの既知のコーディングに違反します
規格。  一方で、これが唯一の現実的な方法であるように思われます。
コードを過度に重複させることなく、これだけ多くのプラットフォームをサポートできます。


## 基本的なガイドライン

これを管理しやすくするために、ほとんどの場合、いくつかのガイドラインに従っています。

「#」ですが、これは避けます。  既知の欠点があります。

それ以外の場合は必要以上に。  同じ理由で、それでも価値がある
通常の`if`ステートメントをインデントします。

`GC_ALLOCD_BYTES_PER_FINALIZER=<value>`- 代替のデフォルト値を設定します
`GC_allocd_bytes_per_finalizer`変数。

`GC_TIME_LIMIT=<milliseconds>`- 代替のデフォルト値を設定します
`GC_time_limit`変数 (これを`GC_TIME_UNLIMITED`に設定すると、基本的に
世代別コレクションを残しつつ増分コレクションを無効にする
有効になります）。

`GC_FULL_FREQ=<n>`- 部分コレクションの代替デフォルト数を設定します
完全な収集の間 (増分収集モードの場合にのみ問題となる)
がオンになっています）。

`NO_CANCEL_SAFE`(スレッドを備えた POSIX プラットフォームのみ) - 心配する必要がなくなります。
スレッドのキャンセルに対してコレクターを安全にします。  キャンセルがあったと仮定すると、
クライアントでは使用されません。  (とにかくキャンセルを使用すると、スレッドが終了する可能性があります
予期せぬところでキャンセルされる場合があります。） このオプションを使用しなくても、
`PTHREAD_CANCEL_ASYNCHRONOUS`はコレクターにとって決して安全ではありません。  （我々は出来た
回収者なしでその安全性について議論する。)

`UNICODE`(Win32 のみ) - Unicode バリアント (サフィックス「W」付き) の使用を強制します。
ANSI/ASCII API ではなく Win32 API (サフィックス「A」付き)。  役に立つ
WinCE。

`HOST_ANDROID`(または`__ANDROID__`) - Android NDK のターゲット コンパイル
プラットフォーム。

`SN_TARGET_PS3`- Sony PS/3 のコンパイルをターゲットとします。

`USE_GET_STACKBASE_FOR_MAIN`(Linux のみ) - 強制的に使用します
`__libc_stack_end`の代わりに`pthread_attr_getstack()`(または任意のコードの代わりに)
ハードコードされた値) を使用して、原始スレッドのスタックの一番下を取得します (次の場合に便利です)
クライアントはプログラムのアドレス空間を変更します)。

`EMSCRIPTEN_ASYNCIFY`(Emscripten のみ) - 非同期機能の使用を強制します。
ほとんどの場合、これは Emscripten の比較的まれに使用される機能であることに注意してください。
開発者はこれを使用しておらず、中規模から大規模までうまく拡張できません。
コードベース。  ただし、この機能は少なくとも`gctest`を渡すために必要です。  必要
`-sASYNCIFY`リンカー フラグ。
